# SO-101 세팅 가이드

이 문서는 SO-101 리더/팔로워 로봇팔을 LeRobot으로 세팅하고, 캘리브레이션과 기본 텔레오퍼레이션을 실행하는 방법을 정리한 팀용 가이드입니다.

## 1. 폴더 구조

졸업과제 작업은 아래 폴더에서 진행합니다.

```text
~/capstone-sim2real
```

LeRobot은 외부 도구처럼 따로 둡니다.

```text
~/lerobot
```

LeRobot 캘리브레이션 파일은 프로젝트 폴더가 아니라 아래 경로에 저장됩니다.

```text
~/.cache/huggingface/lerobot/calibration/
```

SO-101 실행 스크립트는 프로젝트 안에 있습니다.

```text
~/capstone-sim2real/scripts
```

## 2. 기본 환경 켜기

직접 LeRobot 명령어를 실행할 때는 먼저 가상환경을 켭니다.

```bash
cd ~/lerobot
source .venv/bin/activate
```

설치 확인:

```bash
python - <<'PY'
import lerobot, serial
print("lerobot", lerobot.__version__)
print("pyserial", serial.VERSION)
PY
```

프로젝트 스크립트는 내부에서 자동으로 `~/lerobot/.venv`를 불러오므로 보통은 직접 `source` 하지 않아도 됩니다.

## 3. USB 포트

현재 장비에서 쓰는 안정적인 USB serial 경로입니다.

```text
리더:
/dev/serial/by-id/usb-1a86_USB_Single_Serial_5AE6085435-if00

팔로워:
/dev/serial/by-id/usb-1a86_USB_Single_Serial_5AE6086462-if00
```

장치 확인:

```bash
ls -l /dev/serial/by-id/* /dev/ttyACM*
lsusb
```

주의: `/dev/ttyACM0`, `/dev/ttyACM1` 번호는 재부팅이나 재연결 후 바뀔 수 있습니다. 코드나 명령어에는 가능하면 `/dev/serial/by-id/...` 경로를 사용합니다.

## 4. 모터 응답 확인

USB serial이 잡혀도 모터 전원이나 서보 케이블에 문제가 있으면 모터가 응답하지 않을 수 있습니다. 텔레오퍼레이션이 안 되면 먼저 모터 스캔을 합니다.

간단 실행:

```bash
cd ~/capstone-sim2real
./scripts/so101_scan_motors.sh
```

정상 결과는 각 팔에서 모터 ID 1-6이 모두 보이는 것입니다.

```text
{1000000: [1, 2, 3, 4, 5, 6]}
```

결과가 `{}` 이거나 일부 ID가 빠지면 아래를 확인합니다.

```text
1. 로봇팔 전원
2. 서보 데이지체인 케이블
3. USB serial 어댑터 연결
4. 보드와 첫 번째 모터 사이 케이블
5. 다시 모터 스캔
```

## 5. 캘리브레이션 다시 하기

기존 캘리브레이션을 지우고 다시 하려면 아래 순서로 실행합니다.

```bash
cd ~/capstone-sim2real
./scripts/so101_reset_calibration.sh
./scripts/so101_calibrate_leader.sh
./scripts/so101_calibrate_follower.sh
```

각 캘리브레이션에서 해야 할 일:

1. 팔을 관절 범위의 대략 중간 자세로 둡니다.
2. Enter를 누릅니다.
3. `wrist_roll`을 제외한 모든 관절을 천천히 전체 범위로 움직입니다.
4. 다시 Enter를 눌러 저장합니다.

직접 명령어로 실행하려면 아래를 사용합니다.

리더 캘리브레이션:

```bash
lerobot-calibrate \
  --teleop.type=so101_leader \
  --teleop.port=/dev/serial/by-id/usb-1a86_USB_Single_Serial_5AE6085435-if00 \
  --teleop.id=my_leader
```

팔로워 캘리브레이션:

```bash
lerobot-calibrate \
  --robot.type=so101_follower \
  --robot.port=/dev/serial/by-id/usb-1a86_USB_Single_Serial_5AE6086462-if00 \
  --robot.id=my_follower
```

## 6. 리더-팔로워 텔레오퍼레이션

30 Hz 실행:

```bash
cd ~/capstone-sim2real
./scripts/so101_teleop_30.sh
```

60 Hz 실행:

```bash
cd ~/capstone-sim2real
./scripts/so101_teleop_60.sh
```

멈출 때는 터미널에서 `Ctrl+C`를 누릅니다.

30 Hz와 60 Hz 차이:

```text
30 Hz: 더 안정적이고 통신 부담이 적음
60 Hz: 반응이 더 빠르고 부드러울 수 있음
```

카메라 녹화나 데이터 저장까지 같이 할 때는 30 Hz가 더 안전합니다. 단순 리더-팔로워 테스트는 60 Hz도 가능합니다.

직접 명령어로 실행하려면 아래를 사용합니다.

```bash
lerobot-teleoperate \
  --robot.type=so101_follower \
  --robot.port=/dev/serial/by-id/usb-1a86_USB_Single_Serial_5AE6086462-if00 \
  --robot.id=my_follower \
  --teleop.type=so101_leader \
  --teleop.port=/dev/serial/by-id/usb-1a86_USB_Single_Serial_5AE6085435-if00 \
  --teleop.id=my_leader \
  --fps=30
```

## 8. 자주 나는 문제

### USB serial은 보이는데 모터가 안 보임

증상:

```text
Full found motor list: {}
```

또는:

```text
Missing motor IDs
```

원인 후보:

```text
1. 로봇팔 전원이 안 들어옴
2. 서보 케이블이 빠짐
3. USB serial 어댑터는 잡혔지만 모터 버스가 끊김
4. 특정 모터 ID만 응답하지 않음
```

해결 순서:

```bash
cd ~/capstone-sim2real
./scripts/so101_scan_motors.sh
```

모터 ID 1-6이 모두 보이면 다시 텔레오퍼레이션을 실행합니다.

### 첫 실행은 실패하고 두 번째 실행은 되는 경우

가끔 시작할 때 특정 서보가 한 번 응답하지 않을 수 있습니다. 모터 스캔 후 다시 실행하면 정상 동작하는 경우가 있습니다.

### 포트 번호가 바뀐 경우

`/dev/ttyACM0`, `/dev/ttyACM1` 번호는 바뀔 수 있습니다. 아래처럼 by-id 경로를 확인합니다.

```bash
ls -l /dev/serial/by-id/*
```

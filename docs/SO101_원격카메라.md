# SO-101 원격 카메라 연결 가이드

Jetson에 연결된 USB 카메라 영상을 Windows PC에서 GStreamer로 실시간 확인하는 방법입니다.

## 1. 전제

```text
Jetson: USB 카메라 2대 연결
PC: Windows
방식: GStreamer UDP/RTP 송출
사용 카메라 노드: /dev/video0, /dev/video2
```

Jetson과 Windows PC는 같은 네트워크에 있어야 합니다.

## 2. 카메라 확인

Jetson에서 실행:

```bash
ls /dev/video*
```

보통 아래처럼 잡힙니다.

```text
/dev/video0
/dev/video1
/dev/video2
/dev/video3
```

일반적으로 실제 영상 노드는 `/dev/video0`, `/dev/video2`입니다. `/dev/video1`, `/dev/video3`은 보조 노드일 수 있습니다.

LeRobot 환경에서 확인하려면:

```bash
cd ~/lerobot
source .venv/bin/activate
lerobot-find-cameras
```

## 3. Windows PC IP 확인

Windows PowerShell 또는 CMD에서 실행:

```powershell
ipconfig
```

현재 네트워크 어댑터의 IPv4 주소를 확인합니다.

예시:

```text
192.168.0.20
```

아래 Jetson 송출 명령의 `PC_IP`를 이 값으로 바꿉니다.

Tailscale로 연결해서 사용할 경우에는 일반 Wi-Fi/LAN IP 대신 Windows PC의 Tailscale IP를 사용합니다.

Windows에서 확인:

```powershell
tailscale ip -4
```

예시:

```text
100.64.12.34
```

이 경우 Jetson 송출 명령에는 아래처럼 넣습니다.

```bash
udpsink host=100.64.12.34 port=5000 sync=false async=false
```

Tailscale IP를 쓰려면 Jetson과 Windows PC가 모두 같은 Tailscale 네트워크에 접속되어 있어야 합니다.

## 4. 카메라 1대 송출

### 4.1 Jetson에서 송출

`PC_IP`를 Windows PC IPv4 주소로 바꿔서 실행합니다.

```bash
gst-launch-1.0 -v \
v4l2src device=/dev/video0 do-timestamp=true ! \
'image/jpeg,width=640,height=480,framerate=30/1' ! \
jpegdec ! videoconvert ! \
x264enc tune=zerolatency speed-preset=ultrafast bitrate=2500 key-int-max=15 ! \
h264parse ! rtph264pay config-interval=1 pt=96 ! \
udpsink host=PC_IP port=5000 sync=false async=false
```

예시:

```bash
gst-launch-1.0 -v \
v4l2src device=/dev/video0 do-timestamp=true ! \
'image/jpeg,width=640,height=480,framerate=30/1' ! \
jpegdec ! videoconvert ! \
x264enc tune=zerolatency speed-preset=ultrafast bitrate=2500 key-int-max=15 ! \
h264parse ! rtph264pay config-interval=1 pt=96 ! \
udpsink host=192.168.0.20 port=5000 sync=false async=false
```

### 4.2 Windows에서 수신

PowerShell 또는 CMD에서 한 줄로 실행합니다.

```powershell
gst-launch-1.0 -v udpsrc port=5000 caps="application/x-rtp,media=(string)video,encoding-name=(string)H264,payload=(int)96,clock-rate=(int)90000" ! rtpjitterbuffer latency=0 ! rtph264depay ! h264parse ! avdec_h264 ! videoconvert ! autovideosink sync=false
```

정상이라면 Windows에 영상 창이 뜹니다.

## 5. 카메라 2대 송출

포트는 카메라별로 다르게 씁니다.

```text
/dev/video0 -> UDP 5000
/dev/video2 -> UDP 5002
```

### 5.1 Jetson 터미널 1

```bash
gst-launch-1.0 -v \

'image/jpeg,width=640,height=480,framerate=30/1' ! \

h264parse ! rtph264pay config-interval=1 pt=96 ! \
udpsink host=PC_IP port=5000 sync=false async=false
```

### 5.2 Jetson 터미널 2

```bash
gst-launch-1.0 -v \

h264parse ! rtph264pay config-interval=1 pt=96 ! \

```

### 5.3 Windows 터미널 1


## 6. 종료

Jetson 송출 터미널과 Windows 수신 터미널에서 `Ctrl+C`를 누릅니다.

## 7. 트러블슈팅

### 7.1 Jetson에서 `gst-launch-1.0`이 없음

Jetson에서 설치:

```bash
sudo apt update
sudo apt install gstreamer1.0-tools \
  gstreamer1.0-plugins-base \
  gstreamer1.0-plugins-good \
  gstreamer1.0-plugins-bad \
  gstreamer1.0-plugins-ugly \
  gstreamer1.0-libav
```

`x264enc` 확인:

```bash
gst-inspect-1.0 x264enc
```

### 7.2 Windows에서 `gst-launch-1.0`이 없음

Windows에 GStreamer를 설치한 뒤 확인합니다.

```powershell
where.exe gst-launch-1.0
gst-launch-1.0 --version
```

명령어가 안 잡히면 GStreamer의 `bin` 폴더를 PATH에 추가하고 PowerShell/CMD를 새로 엽니다.

### 7.3 Windows에서 명령어 문법 에러

Windows에서는 우선 한 줄 명령어를 사용합니다. Linux 줄바꿈 문자 `\`를 Windows에 그대로 붙여 넣으면 에러가 날 수 있습니다.

### 7.4 수신 창이 안 뜸

확인할 것:

```text
1. Jetson 송출 명령의 host=PC_IP가 Windows PC IPv4 주소인지 확인
2. Jetson 송출 포트와 Windows 수신 포트가 같은지 확인
3. Windows 방화벽에서 UDP 5000/5002 허용
4. Jetson과 Windows가 같은 네트워크인지 확인
5. /dev/video0, /dev/video2가 실제 영상 노드인지 확인
```

### 7.5 화면이 검게 나옴

확인할 것:

```text
1. /dev/video1, /dev/video3 대신 /dev/video0, /dev/video2 사용
2. 카메라 하나만 연결해서 테스트
3. 해상도/FPS 낮추기
4. USB 허브 대신 Jetson 본체 포트 사용
```

정지 이미지로 먼저 확인:

```bash
sudo apt install fswebcam
fswebcam -d /dev/video0 -r 640x480 --skip 30 --no-banner test0.jpg
fswebcam -d /dev/video2 -r 640x480 --skip 30 --no-banner test2.jpg
```

### 7.6 `nvv4l2h264enc` 에러

이 문서는 NVIDIA 하드웨어 인코더 대신 `x264enc`를 사용합니다. 아래 에러가 떠도 이 문서의 명령어에는 영향이 없습니다.

```text
No such element or plugin 'nvv4l2h264enc'
```

### 7.7 DeepStream 또는 Argus 경고

USB 카메라는 `v4l2src`를 사용합니다.

```bash
v4l2src device=/dev/video0
```

DeepStream 플러그인 경고나 Argus 경고가 떠도 영상이 정상적으로 뜨면 무시해도 됩니다.

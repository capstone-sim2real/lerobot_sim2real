# SO-101 원격 조작 가이드

이 문서는 리더 팔 없이 SSH로 Jetson에 접속해서 팔로워 로봇팔만 키보드로 움직이는 방법을 정리한 가이드입니다.

## 1. 사용 상황

```text
조작자: 다른 공간의 PC
접속: SSH 또는 Tailscale SSH
실행 위치: Jetson
조작 대상: SO-101 follower
조작 방식: 키보드 입력
```

원격 조작 전에는 카메라로 팔 주변을 확인하는 것을 권장합니다. 원격 카메라 설정은 [SO101_원격카메라.md](./SO101_원격카메라.md)를 참고합니다.

## 2. 모터 응답 확인

먼저 Jetson에 SSH로 접속한 뒤 모터가 정상 응답하는지 확인합니다.

```bash
cd ~/capstone-sim2real
./scripts/so101_scan_motors.sh
```

정상 결과는 모터 ID 1-6이 모두 보이는 것입니다.

```text
{1000000: [1, 2, 3, 4, 5, 6]}
```

결과가 `{}` 이거나 일부 ID가 빠지면 전원, 서보 케이블, USB serial 연결을 먼저 확인합니다.

## 3. 키보드 조작 실행

```bash
cd ~/capstone-sim2real
./scripts/so101_keyboard_control.sh
```

실행하면 현재 팔로워 자세를 시작 자세로 저장하고, 그 자세를 기준으로 제한 범위 안에서만 움직입니다.

## 4. 키 조작

```text
q/a  shoulder_pan   +/-
w/s  shoulder_lift  +/-
e/d  elbow_flex     +/-
r/f  wrist_flex     +/-
t/g  wrist_roll     +/-
y/h  gripper        +/-

space  현재 목표 위치 출력
0      현재 실제 위치를 새 목표로 재설정
?      도움말
x      종료
```

종료할 때는 기본적으로 시작 자세로 돌아간 뒤 연결을 끊습니다.

## 5. 기본 안전값

```text
기본 이동량: 키 한 번당 2도
기본 관절 제한: 시작 자세 기준 ±60도
기본 그리퍼 제한: 시작 값 기준 ±50
```

`±60도`는 SO-101 하드웨어의 원래 한계가 아니라 원격 키보드 조작 중 과하게 움직이지 않도록 넣은 안전 기본값입니다.

안전 한도에 걸리면 목표값은 더 이상 바뀌지 않고 화면에 `limit`가 표시됩니다.

## 6. 옵션

더 천천히 움직이기:

```bash
./scripts/so101_keyboard_control.sh --step 1 --gripper-step 1
```

시작 자세 기준 이동 한도 줄이기:

```bash
./scripts/so101_keyboard_control.sh --max-delta 30
```

시작 자세 기준 이동 한도 늘리기:

```bash
./scripts/so101_keyboard_control.sh --max-delta 90
```

종료할 때 시작 자세로 돌아가지 않기:

```bash
./scripts/so101_keyboard_control.sh --no-return-start
```

도움말 보기:

```bash
./scripts/so101_keyboard_control.sh --help
```

## 7. 좌표 기준 조작

관절 각도 대신 그리퍼 끝 좌표를 기준으로 움직이려면 `--cartesian` 옵션을 사용합니다.

```bash
./scripts/so101_keyboard_control.sh --cartesian
```

좌표 기준 조작은 SO-101 URDF와 LeRobot kinematics 의존성이 필요합니다.

기본 URDF 검색 경로:

```text
~/capstone-sim2real/third_party/SO-ARM100/Simulation/SO101/so101_new_calib.urdf
~/third_party/SO-ARM100/Simulation/SO101/so101_new_calib.urdf
```

다른 위치에 URDF가 있으면 직접 지정합니다.

```bash
./scripts/so101_keyboard_control.sh --cartesian \
  --urdf-path ~/third_party/SO-ARM100/Simulation/SO101/so101_new_calib.urdf
```

LeRobot kinematics 의존성이 없으면 아래처럼 설치합니다.

```bash
cd ~/lerobot
source .venv/bin/activate
pip install -e '.[kinematics]'
```

좌표 모드 키 조작:

```text
w/s  x +/-
a/d  y +/-
r/f  z +/-
y/h  gripper +/-

space  현재 목표 관절값과 목표 XYZ 출력
0      현재 실제 위치를 새 목표로 재설정
?      도움말
x      종료
```

기본 좌표 이동량은 키 한 번당 `0.005 m`입니다.

더 천천히 움직이기:

```bash
./scripts/so101_keyboard_control.sh --cartesian --xyz-step 0.002
```

좌표 모드는 현재 그리퍼 위치 기준 `±0.08 m` 범위 안에서만 목표 좌표를 바꿉니다. 처음 사용할 때는 카메라를 보면서 작은 `--xyz-step` 값으로 테스트합니다.

## 8. 권장 실행 순서

```bash
cd ~/capstone-sim2real
./scripts/so101_scan_motors.sh
./scripts/so101_keyboard_control.sh --step 1
```

처음에는 `--step 1`로 천천히 움직여 보고, 문제가 없으면 기본값으로 실행합니다.

좌표 기준 조작을 테스트할 때:

```bash
./scripts/so101_keyboard_control.sh --cartesian --xyz-step 0.002
```

## 9. 주의사항

```text
1. 팔 주변에 물체가 없는지 확인
2. 카메라로 팔 상태를 보고 조작
3. 이상 동작 시 x 또는 Ctrl+C로 종료
4. 무리하게 큰 --max-delta 값으로 시작하지 않기
5. 로봇팔이 움직이지 않으면 모터 스캔부터 다시 확인
6. 좌표 모드는 IK 결과가 튈 수 있으므로 처음에는 작은 이동량으로 테스트
```
